# Guia de Contribuição e Boas Práticas

Este documento contém todas as orientações para participação no projeto, incluindo o guia de contribuição para alunos e um checklist completo de boas práticas.

---

##  Guia de Contribuição 

Obrigado por contribuir com este projeto! Para garantir organização, qualidade e consistência entre todas as entregas, siga as diretrizes abaixo antes de enviar qualquer atividade, código ou documentação.

### Como contribuir

1. **Leia toda a documentação antes de começar**  
   Certifique-se de entender o propósito do exercício, requisitos técnicos, estrutura dos arquivos e o fluxo sugerido.

2. **Use a estrutura de pastas indicada no projeto**  
   Exemplo:
   ```
   /src
   /docs
   /assets
   ```

3. **Siga os Style Guides definidos**  
   - Nomeação coerente de arquivos e variáveis  
   - Organização dos imports  
   - Comentários claros e úteis  
   - Padrões de HTML, CSS e JavaScript definidos em aula  

4. **Descreva suas alterações**  
   - O que você mudou  
   - Por que você mudou  
   - Pontos de atenção

5. **Documente sempre que criar algo novo**  
   Atualize README, wiki ou notas técnicas.

6. **Peça revisão antes da entrega final**  
   Trabalhos colaborativos devem ser revisados por pelo menos um colega.

7. **Entregas devem ser pontuais**

---

##  Checklist de Boas Práticas

###  Organização
- [ ] Estrutura de pastas correta  
- [ ] Nomes de arquivos padronizados  
- [ ] Sem arquivos temporários ou lixo  

### Código
- [ ] Código funcionando  
- [ ] Sem erros no console  
- [ ] Variáveis e funções claras  
- [ ] Sem trechos comentados desnecessários  

###  HTML/CSS/JS (se aplicável)
- [ ] HTML semântico  
- [ ] CSS organizado  
- [ ] JS limpo e com funções pequenas  
- [ ] Responsividade testada  

###  Documentação
- [ ] README atualizado  
- [ ] Passo a passo testado  
- [ ] Instruções claras para replicação  
- [ ] Decisões técnicas explicadas  

###  Boas práticas gerais
- [ ] Dados sensíveis não incluídos  
- [ ] Imagens adicionadas quando necessário  
- [ ] Mensagens de commit claras  

###  Entrega
- [ ] Testei o fluxo como usuário final  
- [ ] Arquivo entregue no formato exigido  
- [ ] Conferência com a rubrica da atividade
